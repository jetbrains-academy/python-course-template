This type of task allows you to validate the student's code by checking the actual output against the expected output using stdin/stdout. It’s a good choice for simple exercises where writing tests isn’t necessary.

You don’t need to write any test cases. The `test` folder contains two files:
- `input.txt` — The input data that will be passed to the learner's solution.
- `output.txt` — The expected result the program should output.

Check out the [Output task](https://plugins.jetbrains.com/plugin/10081-jetbrains-academy/docs/educator-start-guide.html#output_task) section in the Course creator start guide for more details.
